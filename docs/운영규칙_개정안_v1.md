# CM기획팀 운영 규칙 개정안 (v1)

작성일: 2026-07-07 | 근거: 팀 관리 시트 전수 분석 (이슈 204건, 과업 104건, 전달사항 19건)

---

## 1. 개정 배경

현행 규칙은 시트 공지(전달사항)로 전달되고 수기 준수에 의존하여, 19건 중 14건이 규칙 교정성 공지로 소모되고 있음. 주요 실측 지표:

- 이슈 재질의율 25% (GO*2 이상 52건/204건) — 질의·의견 혼재가 원인 [출처: WBS별 이슈]
- End(real) 입력률 55% (완료 80건 중 53건... 미입력 시 계획 정확도 데이터 축적 불가) [출처: WBS별 진행 현황]

개정 원칙: **규칙은 공지가 아니라 시스템 제약으로 강제한다.**

---

## 2. 개정 사항

### 개정 1. 이슈 워크플로 (현행 Note 5, 6, 10, 13, 15, 16, 18번 통합·대체)

| 항목 | 현행 | 개정 |
|------|------|------|
| 안건 | 자유 기술 → 안건·문제점 혼용 | **"왜 논의가 필요한가" 1문장**으로 형식 고정 |
| 질의 | 본인생각과 혼재 | **번호 목록**으로 분리 입력 (1. 2. 3.) |
| 본인생각 | 질의와 혼재 | **질의와 1:1 대응** 번호 목록, 필수 입력 |
| GO 요청 | 수동 'GO' 입력, 이후 수정 금지(수기) | GO 제출 시 **해당 이슈 자동 잠금** (시스템 강제) |
| 추가 의견 | 시트 댓글 | **사이트 내 댓글** (개정 4) |
| 관리 지표 | 없음 | 재질의율 측정, 목표 25% → 15% |

### 개정 2. 완료 처리 규칙 (현행 Note 11, 12번 대체)

- Start, End(plan): 최초 입력 후 **시스템상 수정 불가** (현행 규칙 유지, 강제 수단만 변경).
- End(real): "지연/조기 시 입력" 조건부 규칙 폐지 → **진행률 100% 입력 시 무조건 필수** (미입력 시 완료 처리 불가). 지연/조기 여부는 시스템이 End(plan)과 비교해 자동 판정·집계.

### 개정 3. 일일 기록 (변경 없음 — 명문화만)

- 회사 ERP 입력 데이터를 원본으로 유지. 사이트는 **조회·집계 전용** (파일 업로드로 갱신).
- 사이트에서 팀원별·프로젝트별·서브코드별 시간 자동 집계 제공 (수기 누계행 폐지).

### 개정 4. 협업 채널 (현행 Note 7, 8, 9, 14번 대체)

- 시트 댓글/메모 → **사이트 내 댓글** (이슈·과업 단위).
- 알림: **Slack 연동** — 다음 이벤트 발생 시 지정 채널로 자동 발송:
  - GO 제출 (→ 피드백 담당자)
  - 피드백 등록 (→ 이슈 작성자)
  - 댓글 @멘션 (→ 해당 팀원)

### 유지 규칙 (개정 없음)

- 매주 월 09:00 주간 계획/실적 회의, 금 17:00 주간 업무 정리 [출처: Note 1, 2번]
- 회의 없는 시간(09~10, 15~17시) / 가능 시간(10~12, 13~15시) [출처: Note 3, 4번]
- 연장근무 신청 당일 16:00까지 [출처: Note 19번]

## 3. 규칙 → 시스템 강제 매핑 (웹사이트 개발 요구사항)

| 규칙 | 웹사이트 강제 기능 |
|------|-------------------|
| 개정 1 질의/의견 분리 | 이슈 등록 폼: 안건(1문장 제한)·질의·본인생각 별도 필드, 필수값 검증 |
| 개정 1 GO 후 잠금 | GO 제출 시 레코드 잠금, 이후 의견은 댓글로만 |
| 개정 2 End(plan) 불변 | 최초 저장 후 해당 필드 수정 UI 비활성 + DB 정책 차단 |
| 개정 2 End(real) 필수 | 진행률 100% 저장 시 End(real) 미입력이면 저장 거부 |
| 개정 3 자동 집계 | 일일 기록 대시보드 (팀원×기간×코드 집계) |
| 개정 4 댓글·알림 | comments 테이블 + Slack Incoming Webhook 연동 |
